프로젝트 빌드도구 설치

  1. cmd를 관리자 권한으로 실행한다.
  2. 창이 열리면 cd project-name을 입력하여 해당 폴더로 이동한다.
  3. 그 후 cmd 창에 npm install --global windows-build-tools을 입력하여 window 환경에서 필요한 빌드툴들을 설치한다.
  4. Successfully Installed Visual Studio Build Tools라고 메세지가 뜨면 성공적으로 설치가 된 것이다.

gulp 설치

  1. cmd를 관리자 권한으로 실행한다.
  2. 창이 열리면 cd project-name을 입력하여 해당 폴더로 이동한다.
  3. gulp를 가장 최신버전인 4.0.X 버전으로 설치한다.
    • 기존 gulp가 있어 최신버전으로 업데이트가 필요한 경우
      • npm install gulp-cli 입력하여 설치한다.
      • 해당 설치가 끝나면 npm install gulp@next --save-dev 입력하여 설치한다.
    • 신규로 설치하는 경우
      • npm install gulp@next --save-dev 입력하여 설치한다.
  4. gulp -v 혹은 gulp --version을 입력한 뒤 설치버전이 확인이 되면 성공적으로 설치가 된 것이다.

스타터사용법

초기설정

  1. SVN에서 체크아웃 받은 프로젝트 폴더를 연다.
  2. 프로젝트 폴더에서 cmd창을 열어 npm install을 입력한다.
  3. 2번 단계를 완료하면 프로젝트 폴더 안 package.json 파일에서 선언된 디펜던시들이 모두 설치된다.
  4. 설치가 완료되면, node_modules 폴더를 삭제하지 않는 이상은 위 과정을 반복할 필요가 없다.

실행방법

  1. SVN에서 체크아웃 받은 프로젝트 폴더를 연다.
  2. 프로젝트 폴더에서 cmd창을 열어 gulp 명령어를 입력하면 브라우저서버가 띄워진다.
  3. 일단 브라우저 서버를 한 번 띄우면, 작업폴더에서 수정할 때마다 브라우저가 새로고침 되면서 변경한 부분을 바로바로 확인할 수 있다.
  4. 위 gulp 명령어는 기본으로 설정된 부분이고, 각 독립적인 태스크를 실행하고자 하려면 하단에 기재된 내용을 입력하면 된다.

코드작성

공통

  • 임의의 축약어를 사용하지 않는다.
  • btn(x) button(O)과 같은 형태로 작성한다.
  • 소문자로 작성한다.

HTML

  • id는 소문자로 시작하는 lowerCamelCase를 사용한다.
  • class는 "-"를 사용하는 kebab-case를 사용한다.
  • 모든 태그와 속성은 소문자로 작성한다.
  • 속성값은 싱글쿼터(')가 아닌 더블쿼터(")를 사용한다.

들여쓰기

  • 4칸의 공백을 가진 tab 1칸을 기본으로 한다.
  • 공백(space bar)은 사용하지 않는다.

주석

  • 주석 내용을 기준으로 좌우 공백 한칸(스페이스)을 사용한다.
  • 레이아웃을 정의하는 주석은 시작과 끝에 추가한다.

BEM

  • Block : 컴포넌트의 최상위 엘리먼트
  • Element : 영역을 구성하는 작은 단위 또는 자식
  • Modifier : 컴포넌트 자식요소의 스타일 또는 액션을 표현

Block : kebab-case 사용

Element : Block + __Element로 작성

Modifier : Block + __Element + --Modifer로 작성

들여쓰기

  • 4칸의 공백을 가진 tab 1칸을 기본으로 하며 공백은 사용하지 않는다.

기타

이미지명

  • 종류-용도-형태-크기로 작성한다.

파일명

  • 파일명은 SB에 지정된 화면아이디를 사용하는 것을 원칙으로 한다.
  • 파일명이 지정되어 있지 않는 경우 kebab-case 를 사용한다.

접근성

접근성
번호 원칙 지침 검사항목 OpenWax KWCAG 2.1
1 인식의 용이성 대체 텍스트 (적절한 대체 텍스트 제공) 텍스트 아닌 콘텐츠는 그 의미나 용도를 인식할 수 있도록 대체 텍스트를 제공해야 한다. O O
2 멀티미디어 대체 수단 (자막 제공) 멀티미디어 콘텐츠에는 자막, 대본 또는 수화를 제공해야 한다 O
3 명료성 (색에 무관한 콘텐츠 인식) 콘텐츠는 색에 관계없이 인식될 수 있어야 한다. O O
4 (명확한 지시 사항 제공) 지시 사항은 모양, 크기, 위치, 방향, 색, 소리 등에 관계없이 인식될 수 있어야 한다. O
5 (텍스트 콘텐츠의 명도 대비) 텍스트 콘텐츠와 배경 간의 명도 대비는 4.5 대 1 이상이어야 한다. O O
6 (자동 재생 금지) 자동으로 소리가 재생되지 않아야 한다. O
7 (콘텐츠 간의 구분) 이웃한 콘텐츠는 구별될 수 있어야 한다. O
8 운용의 용이성 입력장치 접근성 (키보드 사용 보장) 모든 기능은 키보드만으로도 사용할수 있어야 한다. O
9 (초점 이동) 키보드에 의한 초점은 논리적으로 이동해야하며 시각적으로 구별할 수 있어야 한다. O O
10 (조작 가능) 사용자 입력 및 컨트롤은 조작 가능하도록제공되어야 한다. O
11 충분한 시간 제공 (응답시간 조절) 시간제한이 있는 콘텐츠는 응답시간을조절할 수 있어야 한다 O
12 (정지 기능 제공) 자동으로 변경되는 콘텐츠는 움직임을제어할 수 있어야 한다. O
13 광과민성 발작 예방 (깜빡임과 번쩍임 사용 제한) 초당 3~50 회 주기로 깜빡이거나 번쩍이는 콘텐츠를 제공하지 않아야 한다. O
14 쉬운 내비게이션 (반복 영역 건너뛰기) 콘텐츠의 반복되는 영역은 건너뛸수 있어야 한다. O O
15 (제목 제공) 페이지, 프레임, 콘텐츠 블록에는 적절한 제목을 제공해야 한다. O O
16 (적절한 링크 텍스트) 링크 텍스트는 용도나 목적을 이해할 수 있도록 제공해야 한다. O O
17 이해의 용이성 가독성 (기본 언어 표시) 주로 사용하는 언어를 명시해야 한다. O O
18 예측 가능성 (사용자 요구에 따른 실행) 사용자가 의도하지 않은 기능(새 창, 초점에 의한 맥락 변화 등)은 실행되지 않아야 한다. O O
19 콘텐츠의 논리성 (콘텐츠의 선형 구조) 콘텐츠는 논리적인 순서로 제공해야 한다. O
20 (표의 구성) 표는 이해하기 쉽게 구성해야 한다. O O
21 입력 도움 (레이블 제공) 사용자 입력에는 대응하는 레이블을 제공해야 한다. O O
22 (오류 정정) 입력 오류를 정정할 수 있는 방법을 제공해야 한다. O
23 견고성 문법 준수 (마크업 오류 방지) 마크업 언어의 요소는 열고 닫음, 중첩 관계 및 속성 선언에 오류가 없어야 한다. O O
24 웹 애플리케이션 접근성 (웹 애플리케이션 접근성 준수) 콘텐츠에 포함된 웹 애플리케이션은 접근성이 있어야 한다. O
